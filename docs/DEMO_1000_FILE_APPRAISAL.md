# MEMBRA USD 1,000 Demo File Appraisal

This demo assigns illustrative USD values to the repository files that make the MEMBRA landing demo readable, runnable, and deployable.

The matching code file is `examples/membra_1000_file_values.py`.

## File-level appraisal

| File | Layer | USD |
|---|---|---:|
| `public/index.html` | DemoUI | 325 |
| `examples/membra_1000_file_values.py` | PriceOS code demo | 350 |
| `docs/DEMO_1000_FILE_APPRAISAL.md` | Buyer memo | 200 |
| `vercel.json` | Deploy config | 75 |
| `package.json` | Build script | 50 |
| **Total** |  | **1,000** |

## Why these files are valued

`public/index.html` is the public-facing demo. It explains the MEMBRA thesis, shows the appraisal equation, links to core repository artifacts, and gives the project a page that can be deployed.

`examples/membra_1000_file_values.py` is the code-native appraisal proof. It appraises each demo file in dollars and verifies that the bundle totals USD 1,000.

`docs/DEMO_1000_FILE_APPRAISAL.md` is the human-readable memo. It lets a reviewer understand the value allocation without reading code first.

`vercel.json` and `package.json` are small but operationally important. They make the static UI deployable and repeatable.

## Boundary

These are internal demo estimates for presenting the MEMBRA concept. Real market value requires independent validation, usage data, comparable projects, and buyer demand.
