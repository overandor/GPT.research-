## 1. Institutional Valuation Thesis

`overandor` is appraised as a safety-anchor control-plane asset within the Membramoney protocol ecosystem. It does not require standalone revenue to possess economic value because its primary contribution is risk mitigation, standardization, auditability, and capital preservation across dependent systems.

The value is not based on raw lines of code alone. It is based on the system's ability to:

- enforce invariant gates before execution;
- reduce catastrophic execution error probability;
- standardize behavior across dependent repositories;
- create a reliable audit trail;
- preserve capital by preventing loss events;
- reduce switching and integration costs;
- improve institutional diligence readiness.

## 6. Auditor Evidence Requirements

To convert the appraisal into an auditor-grade valuation, attach:

1. Repository map  
   - all repos depending on `overandor`;  
   - import paths;  
   - package references;  
   - CI/deployment links.

2. Invariant gate map  
   - each gate name;  
   - source file path;  
   - test file path;  
   - failure class prevented.

3. Blocked-event ledger  
   - timestamp;  
   - gate triggered;  
   - notional exposure;  
   - action blocked;  
   - avoided-loss estimate.

4. Capital-at-risk model  
   - daily notional monitored;  
   - average trade size;  
   - maximum route size;  
   - wallet/protocol exposure.

5. Auditability proof  
   - logs;  
   - hashes;  
   - Merkle roots, if available;  
   - immutable event exports.

6. Dependency / switching-cost proof  
   - repo count;  
   - integration points;  
   - migration effort estimate;  
   - estimated rebuild timeline.

7. Security posture  
   - secrets scan;  
   - access controls;  
   - signing boundaries;  
   - test coverage;  
   - failure-mode tests.
